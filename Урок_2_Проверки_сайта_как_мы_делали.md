# Урок 2: Какие проверки мы делали (и зачем)

Этот урок объясняет **каждый тип проверки**, который мы выполняли для digitalleague.ru.  
Эти же проверки будут встроены в нашу платформу.

---

## 1. ЗАГОЛОВКИ БЕЗОПАСНОСТИ

**Что это:** HTTP-заголовки, которые сервер отправляет браузеру. Они говорят браузеру, как себя вести.

**Как проверить:** F12 → Network → выбрать запрос → Headers → Response Headers

**Что ищем:**

| Заголовок | Зачем |
|-----------|-------|
| X-Frame-Options | Защита от встраивания сайта в iframe (clickjacking) |
| X-Content-Type-Options: nosniff | Запрет "угадывать" тип файла (MIME-sniffing) |
| Strict-Transport-Security | Принудительный HTTPS |
| Content-Security-Policy | Ограничение источников скриптов (защита от XSS) |

**Если их нет:** Сайт уязвимее к атакам.

---

## 2. КОНТЕНТ БЕЗ JAVASCRIPT

**Что это:** В исходном HTML есть текст и разметка, или только пустой div и скрипты?

**Как проверить:** Ctrl+U (View Page Source) — смотрим, что пришло с сервера.

**Почему важно:**
- Поисковые роботы могут не выполнять JS → не увидят контент
- Пользователи с отключённым JS увидят пустую страницу

**Хорошо:** В HTML есть заголовки, текст, ссылки.  
**Плохо:** Только `<div id="root"></div>` и скрипты.

---

## 3. SITEMAP.XML

**Что это:** Файл со списком страниц сайта для поисковых систем.

**Как проверить:** Открыть `https://сайт.ru/sitemap.xml`

**Ожидаем:** XML с тегами `<url>`, `<loc>` и т.п.  
**Плохо:** HTML (та же страница, что и главная).

**Почему:** Поисковики ожидают XML. Если получают HTML — не могут прочитать карту.

---

## 4. META DESCRIPTION

**Что это:** Тег `<meta name="description" content="...">` в `<head>`.

**Как проверить:** Ctrl+U → Ctrl+F → ищем `meta name="description"`

**Зачем:** Описание под ссылкой в поиске (Google, Yandex). Влияет на клики.

**Если нет:** Поисковик сам подберёт текст — хуже контроль над выдачей.

---

## 5. КОДИРОВКА (TITLE, MANIFEST)

**Что это:** Русский текст должен отображаться правильно, а не как "ÐÐ¸Ð³Ð°".

**Как проверить:** Смотрим title во вкладке браузера и в manifest.json.

**Причина:** Двойная кодировка UTF-8 (текст прочитали как другую кодировку и снова сохранили).

---

## 6. CACHE-CONTROL

**Что это:** Заголовок, который говорит браузеру, как долго хранить страницу в кэше.

**Как проверить:** F12 → Network → Headers → ищем Cache-Control, Expires

**Зачем:** Меньше запросов к серверу = быстрее загрузка при повторных визитах.
