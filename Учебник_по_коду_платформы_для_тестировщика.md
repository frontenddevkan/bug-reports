### 1.2 Технологический стек

| Технология | Для чего |
|---|---|
| **HTML5** | Структура страницы, семантические теги |
| **CSS3** | Оформление, анимации, адаптивность |
| **Vanilla JavaScript** | Логика без фреймворков (React, Vue и т.д. не используются) |
| **Canvas API** | Анимация фоновой сетки с "током" |

| **MediaDevices API** | Захват скриншотов экрана |

### 1.3 Как файлы связаны

```
index.html
  ├── подключает → styles.css  (строка 8: <link rel="stylesheet">)
  └── подключает → app.js      (строка 2158: <script src="...">)
```

**Принцип работы:** это полностью статическое приложение. Нет сервера, нет базы данных. Всё работает в браузере пользователя. Данные сохраняются в `localStorage` — это встроенное хранилище браузера, которое доступно на каждом сайте (до ~5 МБ). Проект размещён на GitHub Pages — бесплатный хостинг для статических сайтов.

### 1.4 Поток данных

```
Пользователь кликает → JavaScript обрабатывает событие →
→ Обновляет DOM (HTML на странице) →
→ Сохраняет состояние в localStorage →
→ При перезагрузке: читает из localStorage → восстанавливает состояние
```

## 2. HTML — скелет страницы (index.html)

HTML (HyperText Markup Language) — язык разметки. Он описывает СТРУКТУРУ страницы: что на ней есть, в каком порядке, как элементы вложены друг в друга.

### 2.1 Метаданные и подключения (строки 1-9)

```html
<!-- строка 1 -->
<!DOCTYPE html>
```
**Объяснение:** Это декларация типа документа. Говорит браузеру: "Это HTML5-документ". Без неё браузер может перейти в "режим совместимости" (quirks mode) и отображать страницу по старым правилам.

```html
<!-- строка 2 -->
<html lang="ru">
```
**Объяснение:** Корневой элемент всей страницы. `lang="ru"` сообщает браузеру и поисковикам, что контент на русском языке. Это влияет на:
- Работу проверки орфографии
- Произношение в скринридерах (для людей с нарушением зрения)
- SEO (поисковая оптимизация)

```html
<!-- строка 3-4 -->
<head>
    <meta charset="UTF-8">
```
**Объяснение:** `<head>` — секция метаданных (не видна на странице). `charset="UTF-8"` — кодировка символов. UTF-8 поддерживает кириллицу, эмодзи, иероглифы — практически все символы мира. Без этой строки русские буквы могут превратиться в "кракозябры".

```html
<!-- строка 5 -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
```
**Объяснение:** Критически важно для мобильных устройств. Без этого тега телефоны отобразят страницу как уменьшенную десктопную версию. `width=device-width` — ширина страницы = ширина экрана устройства. `initial-scale=1.0` — начальный масштаб 100%.

```html
<!-- строка 6 -->
    <title>ТРЕНАЖЕР ТЕСТИРОВЩИКА — симулятор рабочего дня QA</title>
```
**Объяснение:** Заголовок вкладки в браузере. Также используется поисковиками как заголовок в результатах поиска.

```html
<!-- строка 8 -->
    <link rel="stylesheet" href="bug-reports-platform/styles.css">
```
**Объяснение:** Подключение внешнего CSS-файла. `rel="stylesheet"` — тип связи (стили). `href` — путь к файлу. Браузер скачивает и применяет стили ДО отображения страницы — поэтому пользователь не видит "голый" HTML.

### 2.2 Canvas-фон и шапка (строки 10-51)

```html
<!-- строка 11 -->
<canvas id="bgChargeCanvas" class="bg-charge-canvas" aria-hidden="true"></canvas>
```
**Объяснение:** Элемент `<canvas>` — это "холст", на котором JavaScript рисует графику. Используется для анимации фоновой сетки с бегущими огоньками ("ток по проводам").
- `id="bgChargeCanvas"` — уникальный идентификатор, через который JS находит этот элемент
- `class="bg-charge-canvas"` — CSS-класс для стилизации (position: fixed, растянут на весь экран)
- `aria-hidden="true"` — говорит скринридерам "пропусти этот элемент" (он декоративный)

```html
<!-- строки 12-16 -->
<header class="header">
    <div class="header-left">
        <h1>ТРЕНАЖЕР ТЕСТИРОВЩИКА</h1>
        <p class="subtitle">Инструмент для тестировщиков: теория, тренажёры, баг-репорты...</p>
    </div>
```
**Объяснение:**
- `<header>` — семантический тег HTML5, означающий "шапка сайта"
- `<h1>` — главный заголовок страницы (должен быть один на странице для SEO)
- `<p class="subtitle">` — параграф с классом subtitle для стилизации подзаголовка

```html
<!-- строки 18-27 -->
<a class="header-donate" href="https://boosty.to/eliksir/donate" target="_blank" rel="noopener noreferrer">
    <span class="money-icon"></span>
    <span class="donate-label">Задонатить</span>
</a>
```
**Объяснение:** Ссылка на донат.
- `target="_blank"` — открывает в новой вкладке
- `rel="noopener noreferrer"` — **важно для безопасности!** Без `noopener` открытая страница может получить доступ к объекту `window.opener` и перенаправить вашу вкладку на фишинговый сайт. `noreferrer` не передаёт HTTP-заголовок Referer — защита приватности.
- `<span class="money-icon">` — пустой span, иконка рисуется через CSS (::before или background-image)

### 2.3 Роадмап тестировщика (строки 53-156)

```html
<!-- строка 53 -->
<section class="roadmap-banner">
```
**Объяснение:** `<section>` — семантический блок, объединяющий тематически связанный контент. Это горизонтальная лента с этапами карьеры тестировщика: Trainee → Junior → Middle → Auto QA → Senior.

```html
<!-- строки 62-73 -->
<div class="roadmap-timeline" id="roadmapTimeline">
    <div class="roadmap-timeline-progress" id="roadmapTimelineProgress"></div>
    <div class="roadmap-step roadmap-step--grade" data-step-id="trainee-start">
        <button type="button" class="roadmap-circle">
            <span class="roadmap-circle-index">T</span>
        </button>
        <div class="roadmap-step-labels">
            <div class="roadmap-step-grade">Trainee</div>
        </div>
    </div>
```
**Объяснение:**
- `roadmap-timeline` — контейнер для всех шагов, расположенных горизонтально
- `roadmap-timeline-progress` — полоска прогресса (зелёная), заполняется по мере нажатия на кружки
- `data-step-id="trainee-start"` — **data-атрибут**: пользовательский атрибут для хранения данных. JavaScript использует его, чтобы связать DOM-элемент с состоянием в localStorage
- `<button type="button">` — кнопка-кружок. `type="button"` важен — без него в `<form>` кнопка по умолчанию будет `type="submit"` и отправит форму
- Каждый шаг — кликабельный кружок с номером. При клике JS переключает состояние "пройден/не пройден"

### 2.4 Трёхколоночный layout (строки 158-252)

```html
<!-- строка 159 -->
<section class="my">
    <div class="one">...</div>   <!-- Левая колонка: навигация -->
    <div class="two">...</div>   <!-- Центр: основной контент -->
    <div class="three">...</div> <!-- Правая колонка: теория -->
</section>
```
**Объяснение:** Классическая трёхколоночная раскладка через CSS Flexbox. Класс `.my` — flex-контейнер. `.one` — левый сайдбар (фиксированная ширина), `.two` — занимает всё свободное место (flex: 1), `.three` — правый сайдбар.

#### Левая навигация (.one)

```html
<!-- строки 162-169 -->
<button type="button" class="left-nav-link btn" data-scroll-target="#bugReportsList">Багрепорты</button>
<button type="button" class="left-nav-link btn" data-scroll-target="#testsPanel">Тесты</button>
```
**Объяснение:** Кнопки навигации. `data-scroll-target` хранит CSS-селектор секции, к которой нужно проскроллить (или открыть модалку). JavaScript находит все кнопки с этим атрибутом и вешает обработчики кликов.

#### Центральный контент (.two)

Содержит:
1. **Симуляция рабочего дня** — доска задач в стиле Jira
2. **SDLC-визуализация** — интерактивные этапы жизненного цикла
3. **Кнопка "Создать багрепорт"** — открывает модалку
4. **Список багрепортов** — карточки созданных отчётов

#### Правая колонка (.three) — Бегущая теория

```html
<!-- строки 255-256 -->
<div class="theory-ticker">
    <div class="theory-ticker-inner">
        <p class="theory-block-title">Модель качества ПО: 6 характеристик</p>
        <ul>...</ul>
    </div>
</div>
```
**Объяснение:** `theory-ticker` — контейнер с `overflow: hidden` (скрывает то, что не помещается). `theory-ticker-inner` — внутренний блок, который CSS-анимация `theory-scroll` медленно двигает вверх за 520 секунд. Создаёт эффект "бегущей строки" с теоретическим контентом.

### 2.5 Модальные окна

Модальное окно — это всплывающее окно, которое блокирует взаимодействие с основной страницей.

**Паттерн модалки в проекте:**
```html
<div id="bugReportModal" class="modal hidden">     <!-- Фон-оверлей -->
    <div class="modal-content">                     <!-- Белое окно -->
        <div class="modal-header">                  <!-- Шапка с крестиком -->
            <h2>Заголовок</h2>
            <button class="btn-close">&times;</button>
        </div>
        <form>...</form>                            <!-- Содержимое -->
    </div>
</div>
```

**Как работает:**
1. По умолчанию модалка имеет класс `hidden` → `display: none !important` → невидима
2. При клике JS убирает класс `hidden` → модалка появляется
3. Клик на крестик / фон / Escape → JS добавляет `hidden` обратно

**Модалки в проекте:**
- `bugReportModal` — создание багрепорта (форма с полями Title, Steps, Expected, Actual, Environment, Priority, Severity)
- `docModal` — создание документации (требования, фичи)
- `createChecklistModal` — конструктор чек-листов
- `testsModal` — раздел тестов (квиз)
- `interviewModal` — подготовка к собеседованию (вопрос-ответ)
- `testCasesModal` — тест-кейсы
- `testAssignmentsModal` — тестовые задания
- `checklistsModal` — прохождение чек-листов

### 2.6 Попапы (строки 2124-2151)

```html
<!-- строки 2125-2132 -->
<div id="greetingPopup" class="greeting-popup hidden" aria-live="polite">
```

**Объяснение:** `aria-live="polite"` — атрибут доступности. Скринридер объявит содержимое этого блока, когда он появится, но дождётся паузы (не прерывает пользователя). Попап автоматически скрывается через 4 секунды или при любом взаимодействии.

### 2.7 Подключение скрипта (строка 2158)

```html
<script src="bug-reports-platform/app.js"></script>
```
**Объяснение:** Скрипт подключён В КОНЦЕ body, а не в head. Это важно: к моменту выполнения скрипта все HTML-элементы уже существуют в DOM, и `getElementById` их найдёт. Если поставить скрипт в head — элементы ещё не будут созданы и JS вернёт `null`.

## 3. CSS — визуальный стиль (styles.css)

CSS (Cascading Style Sheets) — язык стилей. Описывает КАК выглядит страница: цвета, размеры, расположение, анимации.

### 3.1 Сброс стилей (строки 12-17)

```css
/* строки 13-17 */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
```
**Объяснение:**
- `*` — универсальный селектор, выбирает ВСЕ элементы на странице
- `margin: 0; padding: 0;` — убирает стандартные отступы. Без этого каждый браузер добавляет свои (разные!) отступы к элементам
- `box-sizing: border-box;` — **критически важное свойство!** По умолчанию `width: 100px` задаёт ширину КОНТЕНТА, а padding и border добавляются сверху. С `border-box` ширина 100px ВКЛЮЧАЕТ padding и border. Без этого вёрстка постоянно "ломается" на 1-2 пикселя.

### 3.2 CSS-переменные и body (строки 19-51)

```css
/* строки 19-51 */
body {
    font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
    --bg-deep: #020617;
    --panel-bg: rgba(15, 23, 42, 0.9);
    --panel-border: rgba(148, 163, 184, 0.6);
    --text-main: #e5e7eb;
    --text-soft: #9ca3af;
    background: ...;
    min-height: 100vh;
    color: var(--text-main);
}
```
**Построчное объяснение:**

- `font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;`
  - Список шрифтов в порядке приоритета. Браузер берёт первый доступный. Segoe UI — шрифт Windows, Tahoma — запасной для Windows, Geneva — для macOS, Verdana — универсальный, sans-serif — системный "без засечек"

- `--bg-deep: #020617;` — **CSS-переменная** (кастомное свойство). `#020617` — почти чёрный с лёгким синим оттенком. Переменные позволяют менять цвет в одном месте — и он обновится везде, где используется `var(--bg-deep)`

- `--panel-bg: rgba(15, 23, 42, 0.9);` — фон панелей. `rgba` — цвет с прозрачностью. R=15, G=23, B=42, A=0.9 (90% непрозрачности, т.е. 10% прозрачности). Через панель слегка просвечивает фон

- `--text-main: #e5e7eb;` — основной цвет текста: светло-серый (почти белый). На тёмном фоне читается хорошо

- `--text-soft: #9ca3af;` — мягкий серый для подсказок и второстепенного текста

- `min-height: 100vh;` — минимальная высота = 100% высоты экрана (viewport height). Страница всегда занимает как минимум весь экран, даже если контента мало

- `color: var(--text-main);` — использование переменной. `var()` подставляет значение переменной

### 3.3 Фоновые градиенты

```css
background:
    radial-gradient(circle at 0% 0%, rgba(56, 189, 248, 0.18), transparent 55%),
    radial-gradient(circle at 100% 100%, rgba(129, 140, 248, 0.24), transparent 55%),
    radial-gradient(circle at 100% 0%, rgba(56, 189, 248, 0.16), transparent 55%),
    var(--bg-deep);
```
**Объяснение:** Несколько слоёв фона, наложенных друг на друга (первый — сверху):
1. Голубое свечение в левом верхнем углу (18% непрозрачности)
2. Фиолетовое свечение в правом нижнем (24%)
3. Голубое свечение в правом верхнем (16%)
4. Тёмный фон `--bg-deep` — самый нижний слой

`radial-gradient(circle at X Y, color1, color2 stop)` — круговой градиент от цвета1 к цвету2. `transparent 55%` — на расстоянии 55% от центра цвет полностью исчезает.

### 3.4 Анимация фона (body::before, body::after)

```css
/* строки 53-66 */
body::before,
body::after {
    content: '';
    position: fixed;
    inset: -20%;
    background: linear-gradient(...);
    opacity: 0.55;
    mix-blend-mode: screen;
    animation: diagonals-slide 30s linear infinite alternate;
}
```
**Объяснение:**
- `::before` и `::after` — **псевдоэлементы**. Это "виртуальные" элементы, которые не существуют в HTML, но CSS создаёт их автоматически. `content: ''` обязателен — без него псевдоэлемент не появится
- `position: fixed; inset: -20%;` — зафиксированы на экране и выходят за его границы на 20% с каждой стороны (чтобы при анимации не было видно краёв)
- `mix-blend-mode: screen;` — режим наложения "экран": цвета складываются, чёрный становится прозрачным, светлые усиливаются. Создаёт эффект мягкого свечения поверх фона
- `animation: diagonals-slide 30s linear infinite alternate;` — анимация движения за 30 секунд, повторяется бесконечно, `alternate` — при каждом повторе направление меняется (вперёд-назад)

### 3.5 Canvas-слой (строки 69-77)

```css
.bg-charge-canvas {
    position: fixed;
    inset: 0;
    width: 100%;
    height: 100%;
    pointer-events: none;
    z-index: 0;
    mix-blend-mode: screen;
}
```
**Объяснение:**
- `position: fixed; inset: 0;` — прибит к экрану, растянут на 100%
- `pointer-events: none;` — **важно!** Клики и касания "проходят сквозь" canvas к элементам под ним. Без этого canvas перехватывал бы все клики, и кнопки под ним не работали бы
- `z-index: 0;` — порядок наложения: canvas под всем контентом
- `mix-blend-mode: screen;` — огоньки на canvas красиво "светятся" поверх тёмного фона

### 3.6 Класс hidden (строки 92-94)

```css
.hidden {
    display: none !important;
}
```
**Объяснение:** Универсальный класс для скрытия элементов. `!important` перебивает любые другие стили. JS добавляет/убирает этот класс для показа/скрытия модалок, попапов и секций.

### 3.7 Трёхколоночный layout (.my)

```css
.my {
    display: flex;
    gap: 1.5rem;
    align-items: flex-start;
}
```

### 4.1 Инициализация страницы (строки 1-6)

**Объяснение:** По умолчанию браузер запоминает позицию скролла и при обновлении страницы возвращает к ней. `scrollRestoration = 'manual'` отключает это — мы будем управлять скроллом сами.

**Зачем:** Хотим, чтобы при каждой загрузке пользователь видел шапку и роадмап, а не оказывался где-то посередине страницы.

**Объяснение:**

### 4.3 Система скриншотов (строки 51-95)

```javascript
// строки 51-89
async function captureScreenshot() {
    if (!navigator.mediaDevices || !navigator.mediaDevices.getDisplayMedia) {
        alert('Ваш браузер не поддерживает захват экрана...');
        return;
    }
    try {
        const stream = await navigator.mediaDevices.getDisplayMedia({ video: true });
        const track = stream.getVideoTracks()[0];
        const imageCapture = new ImageCapture(track);
        const bitmap = await imageCapture.grabFrame();
        // ...
    } catch (err) {
        console.error('Ошибка при захвате экрана', err);
    }
}
```
**Построчное объяснение:**

- `async function` — асинхронная функция. Позволяет использовать `await` внутри — "подожди, пока операция завершится"
- `navigator.mediaDevices.getDisplayMedia({ video: true })` — **Web API для захвата экрана**. Браузер показывает диалог "Выберите, чем поделиться" (весь экран / окно / вкладка). Возвращает поток видео
- `stream.getVideoTracks()[0]` — из потока берём первый (и единственный) видеотрек
- `new ImageCapture(track)` — API для захвата кадров из видеопотока
- `await imageCapture.grabFrame()` — захватываем один кадр (стоп-кадр экрана). Возвращает `ImageBitmap`
- Далее кадр рисуется на `<canvas>`, конвертируется в `dataURL` (base64-строка картинки) и сохраняется в массив `bugArtifacts`

```javascript
// строки 91-95
if (makeScreenshotBtn) {

```
**Объяснение:** Проверяем, что кнопка существует (не `null`), и вешаем обработчик клика. `addEventListener('click', callback)` — "при клике выполни callback".

### 4.5 Работа с localStorage (строки 155-173)

```javascript
// строки 157-165
function loadCustomChecklists() {
    try {
        const raw = localStorage.getItem(CUSTOM_CHECKLISTS_STORAGE_KEY);
        const parsed = raw ? JSON.parse(raw) : {};
        return parsed && typeof parsed === 'object' ? parsed : {};
    } catch (_) {
        return {};
    }
}
```
**Построчное объяснение:**
- `localStorage.getItem(key)` — читает строку из хранилища по ключу. Возвращает `null`, если ключа нет
- `raw ? JSON.parse(raw) : {}` — тернарный оператор: если `raw` не null, парсим JSON-строку в объект; иначе — пустой объект
- `JSON.parse()` — преобразует строку `'{"a":1}'` в объект `{a: 1}`
- `try/catch` — обёртка на случай ошибки (например, если в localStorage лежит битая строка и `JSON.parse` упадёт)
- `catch (_)` — переменная ошибки названа `_`, потому что мы её не используем

```javascript
// строки 167-173

    try {
        localStorage.setItem(CUSTOM_CHECKLISTS_STORAGE_KEY, JSON.stringify(custom));
    } catch (_) { }

```
**Объяснение:** `JSON.stringify(object)` — обратная операция: объект → JSON-строка. `localStorage.setItem(key, value)` — записывает строку в хранилище. `try/catch` — на случай, если localStorage переполнен (лимит ~5 МБ).

#### Построение линий (строки 299-311)

```javascript
function buildLines(max, isVertical) {
    const lines = [];
    let pos = 0;
    let idx = 0;
    while (pos <= max + 1) {
        const spacing = SPACING_PATTERN[idx % SPACING_PATTERN.length];
        const quarter = idx % 4 === 3; // каждая 4-я линия — только 25% длины
        lines.push({ idx, pos, quarter });
        pos += spacing;
        idx++;
    }
    return lines;
}
```
**Объяснение:**
- Строит массив линий от 0 до `max` (ширины/высоты экрана)
- `idx % SPACING_PATTERN.length` — оператор `%` (остаток от деления) зацикливает паттерн: 0,1,2,3,4,5,6,7,0,1,2,...
- `idx % 4 === 3` — каждая 4-я линия (индексы 3,7,11,...) рисуется укороченной (25% длины) — это придаёт сетке визуальный ритм
- Каждая линия — объект `{ idx, pos, quarter }`: индекс, позиция в пикселях, флаг укороченности

```javascript
function drawDot(x, y, intensity) {
    const coreR = 0.75;                    // радиус ядра точки
    const glowR = 4 + 2 * intensity;      // радиус свечения (больше при вспышке)

    const g = ctx.createRadialGradient(x, y, 0, x, y, glowR);
    g.addColorStop(0, `rgba(255,255,255,${0.28 + 0.62 * intensity})`);     // центр — белый
    g.addColorStop(0.25, `rgba(255,255,255,${0.10 + 0.28 * intensity})`);  // ослабевает
    g.addColorStop(0.55, `rgba(56,189,248,${0.05 + 0.20 * intensity})`);   // голубой ореол
    g.addColorStop(1, 'rgba(30,58,138,0)');                                  // край — прозрачный

    ctx.fillStyle = g;
    ctx.beginPath();
    ctx.arc(x, y, glowR, 0, Math.PI * 2);
    ctx.fill();
    // ... маленькое белое ядро ...
}
```
**Объяснение:**
- `createRadialGradient(x1,y1,r1, x2,y2,r2)` — создаёт круговой градиент между двумя окружностями. Здесь от точки (x,y,0) к (x,y,glowR) — от центра к краю
- `addColorStop(позиция, цвет)` — добавляет точку цвета. Позиция от 0 (центр) до 1 (край)
- `intensity` — от 0 до 1, определяет яркость вспышки. При вспышке `intensity` высокий → цвета ярче, ореол шире
- `ctx.arc(x, y, r, 0, Math.PI * 2)` — рисует окружность (полный круг = 2*PI)

#### Ограничение FPS (строки 502-517)

```javascript
const FRAME_MS = 1000 / 30; // 33.3 мс = 30 FPS
function tick(now) {
    if (document.hidden) {
        raf = requestAnimationFrame(tick);
        return;
    }
    if (now - last >= FRAME_MS) {
        last = now;
        render(now / 1000);
    }
    raf = requestAnimationFrame(tick);
}
raf = requestAnimationFrame(tick);
```
**Объяснение:**
- `requestAnimationFrame(callback)` — просит браузер вызвать `callback` перед следующей отрисовкой (обычно 60 раз в секунду). Это эффективнее `setInterval` — браузер сам выбирает оптимальный момент
- `document.hidden` — `true`, если вкладка неактивна. Когда пользователь переключился на другую вкладку, мы пропускаем отрисовку (экономия CPU)
- `now - last >= FRAME_MS` — рисуем только если прошло 33мс (30 FPS). Без этого ограничения на мониторе 144Гц анимация рендерилась бы 144 раза в секунду — лишняя нагрузка

### 4.8 Роадмап (строки 647-709)

```javascript
function initRoadmap() {
    const steps = Array.from(document.querySelectorAll('.roadmap-step'));
    const progressEl = document.getElementById('roadmapTimelineProgress');
    const storageKey = 'simulator.roadmap.v2';
    let state = {};

    // Загрузка состояния из localStorage
    try {
        const raw = localStorage.getItem(storageKey);
        const parsed = raw ? JSON.parse(raw) : {};
        if (parsed && typeof parsed === 'object') state = parsed;
    } catch (_) { }

    function apply() {
        let lastDoneIndex = -1;
        steps.forEach((step, index) => {
            const id = step.getAttribute('data-step-id');
            const done = !!state[id];
            step.classList.toggle('roadmap-step--done', done);
            if (done) lastDoneIndex = index;
        });
        // Прогресс-бар
        let percent = (lastDoneIndex / (steps.length - 1)) * 100;
        progressEl.style.width = `${percent}%`;
    }

    steps.forEach((step) => {
        const circle = step.querySelector('.roadmap-circle');
        circle.addEventListener('click', () => {
            const id = step.getAttribute('data-step-id');
            state[id] = !state[id]; // toggle
            localStorage.setItem(storageKey, JSON.stringify(state));
            apply();
        });
    });
    apply();
}
```
**Ключевые концепции:**
- `!!state[id]` — двойное отрицание. Преобразует любое значение в boolean (`true`/`false`). `!!undefined` → `false`, `!!true` → `true`
- `classList.toggle('class', force)` — если `force` = true, добавляет класс; если false — убирает
- `step.getAttribute('data-step-id')` — читает значение HTML-атрибута `data-step-id`
- `state[id] = !state[id]` — инверсия: если было `true`, станет `false`, и наоборот

- `locked` — флаг-блокировка. После первого клика по ответу ставится `true`, и повторные клики игнорируются (`if (locked) return`). Без этого пользователь мог бы "перещёлкнуть" ответ
- `escapeHtml(opt.why)` — **защита от XSS-атак!** Если текст содержит `<script>alert('hack')</script>`, функция заменит `<` на `&lt;` — и браузер покажет текст как есть, а не выполнит скрипт
- `item.options.find(o => o.ok)` — метод `find` ищет первый элемент, для которого callback вернёт `true`. Находит правильный ответ

### 4.11 Мини-квиз попап (строки 1522-1731)

```javascript
function initQuizPopup() {
    const POPUP_INTERVAL_MS = 30 * 60 * 1000; // 30 минут в миллисекундах
    const QUESTIONS_PER_POPUP = 4;

    let used = []; // индексы уже показанных вопросов

    function pickSessionQuestions() {
        const pool = QUIZ_QUESTIONS.map((_, idx) => idx)
            .filter(idx => !used.includes(idx));
        if (pool.length < QUESTIONS_PER_POPUP) {
            used = []; // сбрасываем, если вопросы кончились
        }
        // Выбираем 4 случайных
        const picked = [];
        while (picked.length < QUESTIONS_PER_POPUP && pool.length) {
            const rndIdx = Math.floor(Math.random() * pool.length);
            const [val] = pool.splice(rndIdx, 1);
            picked.push(val);
            used.push(val);
        }
    }

    function schedule() {
        timerId = setTimeout(startSession, POPUP_INTERVAL_MS);
    }
    schedule();
}
```
**Объяснение:**
- Каждые 30 минут всплывает мини-тест из 4 случайных вопросов
- `used` — массив индексов уже показанных вопросов. Сохраняется в localStorage, чтобы при перезагрузке не показывать те же вопросы
- `pool.splice(rndIdx, 1)` — удаляет элемент из массива и возвращает его. `[val] = ...` — деструктуризация: берём первый элемент из возвращённого массива
- `Math.random()` — случайное число от 0 до 1. `Math.floor(Math.random() * pool.length)` — случайный индекс массива
- `setTimeout(fn, ms)` — однократный вызов через N миллисекунд. В отличие от `setInterval`, вызывается один раз

### 4.12 Приветственный попап (строки 2298-2327)

```javascript
function initGreetingPopup() {
    const popup = document.getElementById('greetingPopup');
    if (!popup) return;

    function hide() {
        if (popup.classList.contains('hidden')) return;
        popup.classList.add('hidden');
    }

    popup.classList.remove('hidden');   // показываем
    setTimeout(hide, 4000);            // через 4с прячем

    function onAnyInput() {
        hide();
        // Удаляем все слушатели — они больше не нужны
        document.removeEventListener('keydown', onAnyInput, true);
        document.removeEventListener('pointerdown', onAnyInput, true);
        document.removeEventListener('mousedown', onAnyInput, true);
        document.removeEventListener('touchstart', onAnyInput, true);
        window.removeEventListener('scroll', onAnyInput, true);
    }
    // Вешаем слушатели на всё: клавиатура, мышь, тач, скролл
    document.addEventListener('keydown', onAnyInput, true);
    document.addEventListener('pointerdown', onAnyInput, true);
    document.addEventListener('mousedown', onAnyInput, true);
    document.addEventListener('touchstart', onAnyInput, true);
    window.addEventListener('scroll', onAnyInput, true);
}
```
**Ключевые концепции:**
- `addEventListener(event, handler, true)` — третий параметр `true` = фаза захвата (capturing). Событие сначала "спускается" от window к элементу (capturing), потом "всплывает" обратно (bubbling). `true` означает "поймать событие на фазе спуска" — это гарантирует, что мы поймаём его раньше любого другого обработчика
- `removeEventListener` — удаляет обработчик. Важно передать ТОЧНО ту же функцию и те же параметры (включая `true`), иначе удаление не сработает
- **Зачем удалять слушатели:** чтобы не вызывать `hide()` повторно при каждом клике после скрытия попапа. Это и оптимизация, и предотвращение лишних проверок

### 4.15 Функция escapeHtml (строки 2461-2469)

```javascript
function escapeHtml(str) {
    if (!str) return '';
    return str
        .replace(/&/g, '&amp;')    // & → &amp;  (амперсанд)
        .replace(/</g, '&lt;')     // < → &lt;   (меньше)
        .replace(/>/g, '&gt;')     // > → &gt;   (больше)
        .replace(/\"/g, '&quot;')  // " → &quot; (кавычка)
        .replace(/'/g, '&#039;');  // ' → &#039; (апостроф)
}
```
**Объяснение:** Функция защиты от XSS. Заменяет спецсимволы HTML на их безопасные "сущности" (entities). После замены `<script>` превращается в `&lt;script&gt;` — браузер покажет это как текст, а не выполнит как скрипт.

**Порядок замен важен!** Сначала `&` → `&amp;`, потому что `&` используется в самих сущностях. Если заменить сначала `<` на `&lt;`, а потом `&` на `&amp;`, получится `&amp;lt;` — двойное экранирование.

## 5. Найденные и исправленные ошибки

### 5.1 Ошибка: `behavior: 'instant' || 'auto'`

**Файл:** `app.js`, строка 5

**Было:**
```javascript
window.scrollTo({ top: 0, left: 0, behavior: 'instant' || 'auto' });
```

**Стало:**
```javascript
window.scrollTo({ top: 0, left: 0, behavior: 'instant' });
```

**Что не так:** Оператор `||` в JavaScript — это логическое ИЛИ. Но он работает не как булев оператор, а как "оператор выбора первого truthy значения":
- `'instant' || 'auto'` → `'instant'` (строка — truthy, поэтому `||` возвращает её, не дойдя до `'auto'`)
- `null || 'auto'` → `'auto'`
- `'' || 'auto'` → `'auto'` (пустая строка — falsy)

Таким образом `'auto'` никогда не использовалось. Код работал, но содержал мёртвую ветку.

### 5.2 Ошибка: `line.half` вместо `line.quarter`

**Файл:** `app.js`, строки 449 и 479

**Было:**
```javascript
const len = line.half ? viewportH * 0.5 : viewportH;
```

**Стало:**
```javascript
const len = line.quarter ? viewportH * 0.25 : viewportH;
```

**Что не так:** Функция `buildLines()` (строка 306) создаёт объекты вида `{ idx, pos, quarter }`. Свойства `half` в них нет. В JavaScript обращение к несуществующему свойству возвращает `undefined`, а `undefined` — falsy. Поэтому условие `line.half ? ...` ВСЕГДА было false, и огоньки на canvas всегда проходили полную длину линии, игнорируя укороченные линии.

**Последствие бага:** Огоньки (точки) на canvas-анимации не учитывали, что каждая 4-я линия укорочена до 25%. Они продолжали двигаться по всей длине экрана, хотя нарисованная линия заканчивалась на четверти.

### 5.3 Дубликат вопроса "Требования"

**Файл:** `app.js`

**Что не так:** В массиве `QUIZ_QUESTIONS` присутствовали два идентичных элемента:

**Исправление:** Удалён второй дубликат.

### 5.4 Скорость бегущего тикера

**Файл:** `styles.css`, строка 2158

**Было:**
```css

```

**Стало:**

## Словарь терминов

| Термин | Объяснение |
|---|---|
| **DOM** | Document Object Model — древовидное представление HTML в памяти браузера. JS работает с DOM |
| **API** | Application Programming Interface — набор функций, предоставленных браузером (или сервером) |
| **IIFE** | Immediately Invoked Function Expression — функция, выполняемая сразу после определения |
| **XSS** | Cross-Site Scripting — атака, внедряющая вредоносный JS через пользовательский ввод |
| **Truthy/Falsy** | В JS: `false`, `0`, `''`, `null`, `undefined`, `NaN` — falsy; всё остальное — truthy |
| **Callback** | Функция, переданная как аргумент другой функции и вызываемая позже |
| **Event listener** | Обработчик событий: функция, вызываемая при наступлении события (клик, нажатие клавиши) |
| **localStorage** | Хранилище пар ключ-значение в браузере. Данные не удаляются при закрытии вкладки |
| **Canvas** | HTML-элемент для рисования 2D/3D графики через JavaScript |
| **Blob** | Binary Large Object — объект для хранения бинарных данных (файлов) в памяти |
