TGWORKSPACE_BOT — ОПИСАНИЕ И ЛОГИКА
===================================

Кратко о боте
-------------
`@tgworkspace_bot` — телеграм‑бот для симулятора тестировщика и трекера рабочего времени:
- помогает фиксировать задачи и время (как лёгкий таск‑/тайм‑трекер),
- будет связан по логике с веб‑платформой Bug Reports Platform,
- в будущем — часть “симулятора рабочего дня тестировщика” (получение задач, напоминания, маленькие теоретические квизы).

Цели бота
---------
1. Дать начинающему тестировщику привычку:
   - фиксировать, чем он занимался,
   - отслеживать прогресс по обучению,
   - видеть, сколько времени реально тратится на практику и теорию.
2. Связать “симулятор” и реальную жизнь:
   - задачи, которые пользователь делает на веб‑платформе, дублируются/логируются в боте;
   - бот может отправлять напоминания и мини‑задания (“разберись с багом”, “пройди 4 вопроса по теории”).
3. Служить частью портфолио:
   - показать на собеседовании связку: веб‑платформа + Telegram‑бот + единая логика работы тестировщика.

Ключевые функции (план)
-----------------------
1. Трекер рабочего времени:
   - `/start` — приветствие, объяснение, как пользоваться ботом и ссылка на веб‑платформу.
   - Команда “Старт рабочего блока” (например, `/work` или кнопка):
     - бот спрашивает тип активности: Теория / Практика / Проект / Собеседование.
     - фиксирует начало с таймштампом.
   - Команда “Стоп” (`/stop`):
     - бот фиксирует конец, рассчитывает длительность,
     - предлагает короткую заметку “что делал”,
     - сохраняет с метками (тип активности, длительность, дата).

2. Журнал задач и достижений:
   - Возможность коротко записать:
     - “какой баг нашёл”,
     - “какое задание по курсу/симулятору выполнил”,
     - “на какой грейд/компетенцию работал” (линк на чекпоинт в роадмапе).
   - Пример: “Сегодня: оформил 3 багрепорта по шаблону, прошёл 10 вопросов по HTTP”.

3. Напоминания и мини‑квизы:
   - Периодические напоминания (по расписанию или по запросу) “пора немножко теории”:
     - бот отправляет 1–3 вопроса по тестированию (выбор из вариантов),
     - пользователь отвечает кнопками,
     - бот даёт разбор правильного ответа.
   - Темы вопросов синхронизированы с веб‑платформой:
     - SDLC, виды тестирования, баг‑репорты, приоритет/серьёзность, матрица компетенций и т.д.

4. Связь с веб‑симулятором (логика на будущее):
   - Каждое “выполнено” по кружкам‑чекпоинтам в роадмапе может:
     - отправлять событие в бота (через API/вебхук),
     - бот фиксирует это в личном “дневнике прогресса”.
   - По запросу из бота (“/progress”) пользователь может получить:
     - краткий отчёт: какие компетенции уже отмечены выполняемыми,
     - сколько времени суммарно отработано,
     - на каком грейде он сейчас сфокусирован (Trainee / Junior / Middle / AutoQA и т.д.).

5. Бизнес‑логика карьерного трекера:
   - Бот хранит (локально или через API) структуру:
     - Грейд (Trainee, Junior, Junior+, Middle, Middle+, Senior, Auto QA уровни),
     - Компетенции (работа с багами, тест‑кейсы, чек‑листы, документация, API, SQL, логирование, снифферы и т.п.),
     - Связка с реальными задачами:
       - “Сделал 3 багрепорта по шаблону” → зачёт компетенции Junior: Работа с дефектами — “умеет оформлять баг‑репорты по шаблону”.
       - “Сам написал чек‑лист для формы логина” → зачёт компетенции Junior+: Работа с чек‑листами — “может написать чек‑лист на простые проверки”.
   - В будущем: рекомендации от бота:
     - “Ты уже закрыла блок X, следующим логично потренировать Y (например, тест‑кейсы или базовый SQL)”.

Описание в одном абзаце
-----------------------
Telegram‑бот `@tgworkspace_bot` — это “личный ассистент тестировщика” в твоём телефоне, который работает в той же фиолетово‑жёлтой палитре, что и веб‑симулятор. Он помогает фиксировать рабочее время и задачи, подсказывает, какие компетенции ты прокачиваешь прямо сейчас, периодически кидает короткие тесты по теории и связывает результаты с большим роадмапом на веб‑странице. В итоге получается связанное пространство: веб‑интерфейс для багрепортов и теории + бот для ежедневных действий, привычек и отслеживания прогресса.
