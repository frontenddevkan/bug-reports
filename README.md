## Bug Reports Platform

Простая платформа для создания багрепортов и обучения тестированию.

- Веб-интерфейс: `bug-reports-platform/index.html`
- Цель: быстро оформлять багрепорты, сохранять их и в будущем отправлять в Jira / YouTrack / Яндекс Трекер / Trac.

---

### Архитектура (план)

1. **Бэкенд (сервер)**  
   - REST API для всех клиентов (веб, Telegram-бот, десктоп, мобильное приложение)  
   - Хранение багрепортов в БД  
   - Генерация PDF / TXT  
   - Интеграции с трекерами (Jira, YouTrack, Яндекс Трекер, Trac)

2. **Веб-интерфейс (SPA)**  
   - Форма создания багрепорта  
   - Просмотр списка багов  
   - Кнопки: экспорт, отправка в трекер  

3. **Telegram-бот**  
   - Создание багрепорта через чат  
   - Приём скриншотов и файлов  
   - Отправка данных на тот же бэкенд

4. **Десктопное приложение (Electron)**  
   - Обёртка над веб-интерфейсом  
   - Дополнительно: захват экрана, быстрый доступ с ПК  

5. **Мобильное приложение (Android)**  
   - Лёгкий клиент (например, на React Native или как PWA)  
   - Использует те же API бэкенда, что и веб/бот/десктоп  

Все клиенты говорят **с одним центральным бэкендом**, чтобы данные и результаты проверок были синхронизированы.

---

### GitHub Pages (бесплатный хостинг)

Чтобы выложить веб-интерфейс (только фронтенд) на GitHub Pages:

1. Создать репозиторий на GitHub, например `bug-reports`.
2. Добавить в него содержимое этой папки (`index.html`, `styles.css`, `app.js` и другие файлы).
3. Сделать commit и push в ветку `main`:
   ```bash
   git add .
   git commit -m "Initial bug reports platform"
   git push origin main
   ```
4. В настройках репозитория GitHub открыть **Settings → Pages** и выбрать:
   - Branch: `main`
   - Folder: `/root` (если `index.html` в корне репо) или `/docs`, если вынесете туда.
5. После сохранения GitHub выдаст URL вида:
   ```
   https://<ваш-логин>.github.io/bug-reports/
   ```
6. Если `index.html` лежит в подкаталоге `bug-reports-platform`, URL будет:
   ```
   https://<ваш-логин>.github.io/bug-reports/bug-reports-platform/
   ```

Подставьте вместо `<ваш-логин>` своё имя пользователя GitHub.

---

### Локальный запуск

Просто откройте файл `bug-reports-platform/index.html` в браузере (двойной клик или через контекстное меню).
